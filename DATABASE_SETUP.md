# TraceLog - Railway PostgreSQL Setup Guide

## Overview

TraceLog requires a PostgreSQL database to store transcripts and user data. This guide walks you through setting up Railway's PostgreSQL service.

---

## Step 1: Create a Railway Account

1. Go to [railway.app](https://railway.app)
2. Click **Sign up** (or Sign in if you have an account)
3. Choose authentication method (GitHub recommended for easy setup)

---

## Step 2: Create a New Project

1. Click **+ New Project** in your Railway dashboard
2. Select **PostgreSQL** from the templates
3. Railway will create a PostgreSQL database service automatically

---

## Step 3: Get Your Connection String

### Method 1: From Railway Dashboard (Recommended)

1. Go to your PostgreSQL service in Railway
2. Click on the **Connect** tab
3. Under "Postgres Connection URL", copy the full connection string
4. It should look like:
   ```
   postgresql://postgres:RANDOM_PASSWORD@containers.railway.app:PORT/railway
   ```

### Method 2: From Environment Variables

1. In your PostgreSQL service, go to the **Variables** tab
2. Look for `DATABASE_URL` - copy its value
3. Or view individual credentials:
   - **PGHOST**: Database host
   - **PGPORT**: Database port (usually 5432)
   - **PGUSER**: Username
   - **PGPASSWORD**: Password
   - **PGDATABASE**: Database name

---

## Step 4: Update Your Local .env.local

Open `d:\TraceLog\.env.local` and update:

```env
# Paste the connection string from Railway
DATABASE_URL=postgresql://postgres:your_password@containers.railway.app:12345/railway

# Generate a random secret (see instructions below)
BETTER_AUTH_SECRET=your_random_32char_secret_here

# Get from Google AI Studio
GEMINI_API_KEY=your_gemini_key_here

# Local development URLs
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=your_nextauth_secret_key
```

---

## Step 5: Generate Secure Secrets

### Generate BETTER_AUTH_SECRET

**Option 1: Using Node.js**
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

**Option 2: Using OpenSSL (Windows PowerShell)**
```powershell
$bytes = [System.Security.Cryptography.RandomNumberGenerator]::GetBytes(32)
[Convert]::ToBase64String($bytes)
```

**Option 3: Quick Workaround**
Use any strong password generator or: `openssl rand -base64 32` (if you have Git Bash installed)

---

## Step 6: Run Database Setup

### On Windows (PowerShell):
```bash
cd d:\TraceLog
.\setup-db.bat
```

### On macOS/Linux:
```bash
cd ~/TraceLog
bash ./setup-db.sh
```

### Manual Setup:
```bash
# Run migrations to create tables
npm run db:migrate

# Create admin user
npm run db:seed
```

---

## Step 7: Verify Connection

Check if everything is working:

```bash
# Open Prisma Studio to view your database
npm run db:studio
```

You should see:
- ✅ `User` table (with one admin user: `admin@tracelog.com`)
- ✅ `Transcript` table (empty initially)

## Troubleshooting

### "Error: connect ECONNREFUSED"
**Problem**: Can't connect to database
**Solution**:
1. Verify DATABASE_URL in `.env.local` is correct
2. Check Railway database is running (green status in dashboard)
3. Check network connection

### "Error: permission denied for schema public"
**Problem**: Database user doesn't have permissions
**Solution**:
1. Railway should handle this automatically
2. Try creating a new PostgreSQL service in Railway
3. Copy the new connection string

### "Error: database "railway" does not exist"
**Problem**: Database name is wrong
**Solution**:
1. Check the database name in your connection string
2. Railway defaults to `railway` - that's correct
3. If using custom database, update `PGDATABASE` in Railway variables

### "Seed failed: Can't find package @/lib"
**Problem**: ts-node can't resolve aliases
**Solution**:
- This has been fixed in the updated seed script
- Run `npm run db:seed` again

## Railway Environment Variables

Railway allows you to configure services directly in their dashboard. To set up environment variables:

1. Go to your Railway project
2. Click on your service
3. Go to **Variables** tab
4. Add/edit variables as needed

**Recommended variables for Railway:**

```env
DATABASE_URL=postgresql://...  # Auto-generated by Railway
BETTER_AUTH_SECRET=your_secret_here
GEMINI_API_KEY=your_gemini_key
NODE_ENV=production
```
